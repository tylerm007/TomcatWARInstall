Users

You can define Authorized Users for projects using the Default Authentication Provider, including login id, password, and roles.

Background

Security addresses:

  1. Authentication is a pluggable mechanism (a Project property) to verify users at login, returning set of roles.

  2. Authorization defines what Resources are visible.

  3. Role Permissions specify row/column filtering for roles

Note such security applies to all end-points, whether your default API (tables, columns and procedures), or Custom APIs.

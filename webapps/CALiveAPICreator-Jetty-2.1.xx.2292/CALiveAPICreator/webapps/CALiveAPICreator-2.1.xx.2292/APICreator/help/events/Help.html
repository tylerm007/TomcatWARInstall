Request Events

A {{brandPrefixName}} Live API can have any number of Request Events, which will be automatically executed at certain points during a request.

There are currently two types of event handlers:

  1. Request Event Handlers are called for every request, right after the request has been received, but before any "serious" work has really begun.

  2. Response Event Handlers are called for every request, right before the final result is sent back to the client (assuming the request was successful).

For example, you might use Request Events to reject all requests on certain resources if they do not use HTTPS.

Or, define Response Events to modify the request response.

Request Events are one of the ways you can use Java Script for implementing behavior.

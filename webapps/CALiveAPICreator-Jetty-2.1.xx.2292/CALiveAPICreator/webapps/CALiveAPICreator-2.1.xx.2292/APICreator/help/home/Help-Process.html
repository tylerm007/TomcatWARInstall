Process Overview

The basic process is as follows:

  1. Connect: point {{productFullName}} to your SQL database. You can then

    1. Use the Data Explorer to browse and update your data

    2. Use the RESTlab to test the default API (end-points for each table, view and stored procedure)


  2. Then, use the {{productSlimName}} to specify

    1. Custom API: define document-model end-points that combine data from multiple tables, and multiple databases

    2. Rules: database integrity with a combination of spreadsheet-like Reactive Rules and server-side JavaScript.

    3. Security: controls end-point access, and row/column security.

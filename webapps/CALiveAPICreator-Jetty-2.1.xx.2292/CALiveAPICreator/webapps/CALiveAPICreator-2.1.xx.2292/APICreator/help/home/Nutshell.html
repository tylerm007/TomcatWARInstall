The Data Explorer provides Master/Detail and Forms Navigation, starting from a Base Table or Resource. It uses your Live API, so Security and Live Logic are applied to all requests.

Live API

You start by creating a Project, where you register your database. Once completed, your default REST API is operational.

Your Resource end points are the tables and Stored Procedures (if any), and can be accessed via the Data Explorer or the REST Lab.

Use the {{productFullName}} for point-and-click definition of Custom Multi-Table RESTful Resource End Points, which provide:
  1. A Custom API, providing multi-table document-oriented JSON results, aliased / projected columns, etc
  2. SQL automation including filters and join automation for related data
  3. Resource Types provide for results using custom SQL or JavaScript


Use the {{productFullName}} to define fine-grained Security, based on these key concepts:
  1. Authentication, which defines a users' valid Roles
  2. Authorization, which defines Role Permissions to table rows and columns
  3. Powerful Global values and objects, used in row filters


Live Logic


Business logic is the set of multi-table computations, validations, and actions (e.g., auditing, cloning, sending mail etc) performed during an update. Logic consists of:
  1. Reactive Expressions - spreadsheet-like expressions that define how data is computed or validated
    (e.g., Derive balance as sum(purchaseorderList.amount_total where paid=false))
    Reactive means the system automatically adjusts data when referenced data is changed,
    so for example the balance is reduced when the order is paid.
  2. Server-side Java Script - the system builds an Object Model with attribute/object accessors, persistence, and events for insert, update and delete.
Accessed via Live Logic in the Navigation Bar to the left, you can achieve enormous savings in work:
  1. First Look - Explore the logic in Demo API
  2. Closer Look - complex examples (e.g. Bill of Materials), and logic patterns

Count


Counts declares a Parent attribute to be maintained as the count of a designated child rows, with an optional qualification to filter child objects. Parent reference names are derived from Foreign Key Relationships.

For example:

Derive orders.item_count as count(lineitemsList)

Counts are

  1. Significantly simpler than events

  2. Efficient - parent data is adjusted via a 1 row update - the system does not issue a select sum() SQL command

  3. Best Practice Counts as existence checks
Condition Syntax:
  • unlike other expressions, the usual return and row. are not required, nor can you reference loadable libary functions
  • specify == or !=
  • logical operators are &&, ||

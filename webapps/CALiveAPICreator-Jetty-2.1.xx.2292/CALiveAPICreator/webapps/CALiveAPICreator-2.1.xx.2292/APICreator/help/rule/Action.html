Events

Events are server-side JavaScript code for business logic, providing access to external / re-usable functions.

Important

Logic is expressed in both JavaScript events, and Reactive Rules.


  1. Think Spreadsheet . to save meaningful amounts of time by using Reactive Logic rather than events.

  2. For example, defining sums (or other rules) is much simpler than writing event handlers.

Using Events

The Logic Engine invokes Table Events on every update, providing key contextual information including:

  1. row - this object provides access to:
    1. attributes for every column, so you can refer to a Order's row.amount_total
    2. Parent role accessors, so you can refer to an Orders' row.customer.balance
    3. Child role accessors, e.g., obtain a collection of row.lineitems (a collection of lineitem rows).
    Rows also provide persistence to read/alter data through LogicContext (see below).

  2. oldRow - attributes for the pre-updated version of the row, so you can detect changes.

  3. LogicContext - provides many services, such verb (insert, update or delete), services to read and write data (including logic enforcement), create new objects, event logging, etc. See here for more information.

  4. SysLogic - system methods, including InsertIntoFrom (auditing and deep copy), Allocation and findWhere (collection filtering)

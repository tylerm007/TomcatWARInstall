Rules: Reactive Logic and JavaScript


What

  1. Rules enforce database integrity on RESTful update requests - Business Rules for derivations, validations, integration, and events.

  2. Rules are specified using
    • Reactive Logic, plus
    • JavaScript events (server side),
      extensible via loadable libraries (disable Short Menus, API Properties).

  3. Reactive Logic is conceptually similar to spreadsheet cell formulas:
    • You bind expressions (functions) to database columns
    • On RESTful updates, (Post, Put, Delete), the system:
      • Watches for changes to referenced data,
      • Reacts to adjust referencing data,
      • which may Chain to other rules per this video.
    • Rules apply to all updates - both tables, and resources.

  4. Reactive Logic has key differences to conventional procedural code:
    • Invocation is automatic - the engine (not your code) calls the logic - it never forgets
    • Automatic Chaining Change Propagation to dependent data - including related tables
    • Automated SQL and transaction bracketing for Chaining to related tables

Why

Reactive eliminates tedious boilerplate code for Watch / React / Chain, Persistence and Transaction Management.

As a result, a small handful of reactive logic is equivalent to many hundreds of lines of conventional code.

What is a user named filter?

These are user defined syntax with specified (exposed) column attribute names that limit how filters are completed. The filter syntax are available for testing but they can be a security problem in production. You can disable the use of filter in the Settings area of API Properties.


Named Filters
.../v1/[Resource]?filter=balance<1000
.../v1/[Resource]?sysfilter=[sysfiltername[modifier]](expression[, expression [, ...]])
.../v1/[Resource]?userfilter=myNamedFilter([namedParameters])


sysfiltername = equal, notequal, less, lessequal, greater, greatrequal, like, notlike
modifier = _or, _uc, _uc_or (use OR instead of AND for multiple expressions, UPPER case text, UPPER case and use OR)
expression = (columnName: value) or (columnName: 'TextValue')
namedParameters = (parameterName: value) or (parameterName: 'TextValue') - as defined in the User Filter User filters allow you to pre-define arbitrarily complex named objects used to filter resource results. See the documentation for additional details.

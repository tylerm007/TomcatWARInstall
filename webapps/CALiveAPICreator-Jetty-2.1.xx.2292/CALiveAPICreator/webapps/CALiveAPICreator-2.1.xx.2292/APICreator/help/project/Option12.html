Permit authorization parameter in URL

If this option is true, then API users may specify their Auth Token on the URL using the …&auth=123456789:1 format.

This may be desirable if users want to make GET requests straight from a browser, or if for whatever reason they cannot set HTTP headers. (POST/PUT/DELETE also support this, but normally this is not useful)

Obviously, this has security implications, since URLs can be remembered by browsers, are typically logged in servers, routers, etc...

Notice: if this option is false, you will not be able to retrieve the contents of blobs from an HTML document using an href, since there is no way for an href attribute to specify a header when retrieving a resource.

Resources

When you connect, the system creates Default Resources (aka End Points) for each table, view and stored procedures. You can control access using Security.

You can also define Custom Resources, e.g. for custom apps, or for data integration Web Services. You can explore the pre-created samples.
To create a resource:
  1. Click "New Resource", and select a table.
  2. Click "New Level", and select a related table
    to create Sub Resources for a tree-like document model.
  3. Use the Attributes tab to specify which columns are included, and their alias names.


Observe:

  • Custom Resources provide a tree-like document model

  • You can choose and rename the objects and attributes, so your API is truly custom, not merely a flat reflection of your relational tables

  • Sub Resources can combine data from multiple data sources using Resource Types (see sample Resources in the Logic Demo project):
    • Sql
    • MongoDB
    • Other RESTful services


  • Logic and Security apply “underneath” the RESTful API, so are enforced for Data Explorer, or any RESTful API call, whether through default or Custom End-points

  • Your API includes Enterprise-class pattern automation::
    • Pagination
    • Optimistic Locking
    • Generated Primary Key Propagation to related child data, etc.


  • No codegen or deployment, so your resources are operational as soon as you click save

  • Test them in the REST Lab

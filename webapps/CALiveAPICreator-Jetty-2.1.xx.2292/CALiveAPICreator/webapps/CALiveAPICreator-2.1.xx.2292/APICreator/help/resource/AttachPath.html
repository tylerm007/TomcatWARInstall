Resource - Attach Path

This field is required for subresources of a Mongo resource.

It defines where, in the parent tree, the result of the subresource should be attached.

Let's assume that the parent resource returns the following tree: 

{
  "company_name": "Acme",
  "budget": 100000
  "departments": [
    {
      "name": "Marketing",
    },
    {
      "name": "Accounting"
    }
  ]
}

Now let's assume that the subresource will add the employees (from e.g. a SQL database) to the departments. In this case, the attach path would be: departments.employees and the resulting tree would look something like: 

{
  "company_name": "Acme",
  "budget": 100000
  "departments": [
    {
      "name": "Marketing",
      "employees": [
        {
          "emp_name": "Jane Doe"
        }
      ]
    },
    {
      "name": "Accounting",
      "employees": [
        {
          "emp_name": "Jane Doe"
        }
      ]
    }
  ]
}
With this attach path, the parent "row" for the subresource will be the department object, and its attributes (in this example, just the "name" attribute) will be available in the join expression.

REST Lab

Use the REST Lab to test your API, without having to write a one-off test program:

  1. Select End Point type: your end points are your Base Tables, Views, Procedures (these are all created automatically), plus your Custom Resources. Choose a type, then choose an End Point.

  2. URL: this is the URL you can use in REST calls, e.g., a mobile app.
    • You can add URL parameters for filter, order, sysfilter, sysorder (and named filters and sorts) - see docs, or the API Docs on the left Navigation Bar - [x] de-select Short Menu if not visible]).

  3. Send: Fill in the request as appropriate, and click the type of call (GET, PUT etc) to run.

    • Put/Post require the Request field. Tip: copy/paste a response row (including @metadata tag, required for optimistic locking), paste it into the Request, change, and click Put.
    • There are various options you can supply on the @metadata tag, as described in Complex Transactions.
The Logs (Left Navigation Bar) will show you the SQLs and Rules resulting from your request.

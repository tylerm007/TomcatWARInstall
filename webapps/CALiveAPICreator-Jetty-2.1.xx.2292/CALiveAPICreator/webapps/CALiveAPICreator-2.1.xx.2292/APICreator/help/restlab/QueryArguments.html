Query Arguments

You can supply arguments in the URL field for filtering, sorting etc. This can be be useful to test what you might use in your application. Note these require proper HTTP escape characters.

URL arguments can include Stored Procedure arguments, such as
?arg.OrderID=10248

More simply, check Args - the system provides the escape characters, so you can specify filters such as
name like 'Acme%' or balance > 999

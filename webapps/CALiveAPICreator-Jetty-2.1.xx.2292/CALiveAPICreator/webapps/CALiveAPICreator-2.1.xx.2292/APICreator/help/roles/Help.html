Role-based Security

Role-based security provides authorization control over what resources, rows, and columns authenticated users can access.

Roles

Users can have multiple roles. For a role, you can define:

  • What Resource endpoints are accessible (either by default, or specific enumeration)

  • Permissions that specify row/column level access for a specific table


Background

Security addresses:

  1. Authentication is a pluggable mechanism (a Project property) to verify users at login, returning set of roles.

  2. Authorization defines what Resources are visible.

  3. Role Permissions specify row/column filtering for roles



As shown above for get, such security applies to all end-points, whether your default API (tables, columns and procedures), or Custom APIs. Actual filtering is performed on the database, where is most efficient.
